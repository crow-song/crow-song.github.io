dao 层、mybatis 及 SqlSession 作用

dao (data access objection 数据访问对象)层需求 ：

  1. 能与数据库交互
  2. 能执行 sql 语句

mybatis ：向 dao 层提供对象 SqlSession

SqlSession 作用：

  1. 向 sql 语句传入参数
  2. 执行 sql 语句
  3. 获取执行 sql 语句的结果
  4. 事务的控制

如何得到 SqlSession：

  1. 通过配置文件获取数据库连接的相关信息
  2. 通过配置信息构建 SqlSessionFactory
  3. 通过 SqlSessionFactory 打开 数据库会话（SqlSession）

log4j 中的代码意义

log4j.appender.Console.layout.ConversionPattern=%d [%t] %-5p [%c] - %m%n

%d 日志生成的时间

%t 是产生日志所处的线程名称

%p 输出日志的级别，%-5p 将占5位字符，不足5位用空格填补，- 指的是在右边补空

%c 你输出日志的包以及类的全名

%m 是你附加的信息

%n 换行

Q&A

Q:Mybatis ORA-00911: 无效字符、批量更新

mybatis 在进行 foreach 批量更新时会报错，但是将 sql 复制到 plsql 能执行

<foreach collection="list" item="list" index="index" open="begin" close=";end;" separator=";">

在 foreach 中添加 open="begin" close=";end;"

Q: bindingexception parameter not found. available parameters are [list,collection]

controller 使用 Obj[] objs 接受数据，再使用 Arrays.asList(objs); 转换为 list

Q:SQL state [99999] ;error code [17059] 无法转换为内部表示

查到入的值与需要设置的bean值类型不匹配

Q:java.sql.SQLException: 无效的列索引

mybatis 注释可能会引起该问题

Q:available parameters are [list,collection ]

Parameter ‘__frch_item_0’ not found. Available parameters are [list]

可能是参数名错误，从 mybatis 查出来的参数名全为大写，如果传入参数的 map key 值有小写则会出现错误

参考：https://www.cnblogs.com/yangh965/p/6544949.html

Q:mybatis 输入 String 参数没有 getter 方法

A:使用 @Param(“参数名”)，在 Mapper.java 的对应方法上，参数前

public List queryData(@Param(“tableName”) String tableName,Page page){

}

如果不行，在 mapper.xml 中使用 _paramter 替代参数占位符
